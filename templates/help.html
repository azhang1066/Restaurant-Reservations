Getting Started

Prerequisites

  • Python 3.11+ — earlier versions are not tested.
  • A .env file in the project root for credentials and settings. The dashboard writes to it automatically when you save Notification Settings, so you don't need to edit it by hand unless you prefer to.
  • A Resy account (for Resy monitoring) — you'll need your API key and auth token from your browser's developer tools.

Running the dashboard

From the project root, either of these commands starts the server:

python -m app python run_dashboard.py

The dashboard is served at http://localhost:5000. On the same local network, you can also open it on your phone using your machine's IP address — for example, http://192.168.1.42:5000.

Adding a Restaurant

URL resolution

Paste a Resy or OpenTable URL into the Resy / OpenTable URL field and click Resolve URL. The app parses the URL to extract the venue slug and, for Resy, looks up the numeric venue ID via the API. The form fields below are auto-populated — you normally don't need to touch them.

Form fields

  • Restaurant Name — display label in the watchlist. Auto-filled from URL resolution but editable.
  • Platform — Resy or OpenTable. Auto-selected based on the URL you pasted.
  • Resy Venue ID / OpenTable Restaurant ID — the numeric ID the API uses to fetch availability. Auto-filled when you resolve a URL with valid Resy credentials set.
  • Party Sizes — one or more guest counts to search for, entered as chips. Type a number and press Enter or comma. The first chip is the highest priority (used for the booking deep link when multiple sizes match). Drag the ↑ / ↓ arrows to reorder.
  • Days and Time Ranges — check the days you want to monitor and set the earliest and latest time for each. Only slots that fall within a window for the correct day of the week are reported.

Click Add Restaurant to save. The card appears in the Watchlist immediately.

Notification Setup

ntfy (recommended — free, no account)

ntfy.sh is an open-source push notification service. There is nothing to sign up for.

  • A random topic name is generated for you the first time you open the dashboard (e.g. resy-notifier-ab12cd). You can change it to anything you like — just keep it unguessable.
  • To receive notifications on your phone, install the ntfy app (iOS or Android), tap Subscribe to topic, and enter your topic name — or tap the Copy link button in the dashboard and paste it into the app.
  • Notifications include a Book Now action that opens the pre-filled booking page.

Pushover (alternative)

Pushover requires a one-time $5 purchase per platform. Create an account at pushover.net, then create an application to get an App Token. Your User Key is on the Pushover dashboard home page. Enter both in Notification Settings and select Pushover as the provider.

Channels

Use the Push notifications and Email notifications toggles to choose which channels fire. Both can be enabled at once. Email requires SMTP settings to be filled in.

Testing

Click Send test notification after saving settings. A test alert is sent immediately — confirm it arrives on your device before relying on the monitor.

Reading the Activity Log

The Activity Log shows the result of every check cycle and every notification sent. Entries are pruned to the most recent 500.

Icon State Meaning
✅ sent A new slot was found and a push/email notification was dispatched.
🔁 skipped A slot was found but you were already notified about it in a previous check — no duplicate alert.
🔍 nothing found The availability API returned no slots matching your criteria for this restaurant and check cycle.
❌ failed The API call threw an error. Check your credentials and network connectivity.

Book Now link

Log entries for sent notifications include a Book Now → pill. Clicking it opens the pre-filled booking page for that slot in a new tab — same URL that was sent in the push notification.

Check Scheduling

Scheduler status pill

The pill in the top-right of the dashboard header shows when the last automatic check ran and how long until the next one. It refreshes every 30 seconds. While a check is in progress the pill shows Checking….

Check Now

The Check Now button triggers an immediate availability check for all enabled restaurants. It is disabled while a check is already running to prevent overlapping requests.

Check interval

The interval between automatic checks is controlled by the CHECK_INTERVAL_MINUTES variable in your .env file (default: 20 minutes). Edit the value there and restart the server to apply the change.

Deep Links

How they work

When a slot is found, the app builds a pre-filled booking URL containing the date, time, and party size:

  • Resy: resy.com/cities/{city}/venues/{slug}?date=YYYY-MM-DD&seats=N
  • OpenTable: opentable.com/r/{slug}?covers=N&dateTime=YYYY-MM-DDTHH:MM

Before the URL is included in a notification, the app sends a quick HEAD request (2-second timeout) to verify the link is reachable.

Fallback behavior

If the HEAD request fails or times out, the notification falls back to the venue's homepage URL instead of a broken deep link.

Test Link button

Each watchlist card has a Test link button. Clicking it calls /api/restaurants/{id}/deep-link for today's date at 19:00 and your top party size, then opens the resulting URL in a new tab so you can confirm the link lands on the correct booking screen.

Tips & Troubleshooting

Resy auth token expiry

Resy auth tokens expire periodically (typically every few weeks). Symptoms: all Resy checks log ❌ failed with a 401 error. Fix: log in to resy.com, open DevTools → Network, make any request, find the Authorization header value, and paste the new token into Resy Credentials → Auth Token in the dashboard.

No slots are found

  • Confirm the Resy/OpenTable venue ID is correct — resolve the URL again if unsure.
  • Check that your day/time ranges include the times you expect. Slots outside the window are silently filtered.
  • Party size must match exactly what the restaurant is releasing. Try adding additional party sizes.
  • Some restaurants only release slots at midnight or at a specific time — use Check Now to poll on demand.

Deduplication — why you won't get spammed

Once a notification is sent for a specific slot (venue + date + time + party size), that slot is recorded in the database. Subsequent checks skip it and log 🔁 skipped. If the slot disappears from the API and later reappears — for example because someone cancelled — the record is evicted and you will be notified again.

Multi-party-size behavior

When you add more than one party size to a restaurant, each size is checked independently. If the same time slot is available for multiple sizes in one check run, only one notification is sent (the first match wins). Across separate check runs, deduplication applies per party size independently.

Tip: Order your party sizes by priority — put your preferred guest count first. The first chip is used as the booking link's party size when multiple sizes match the same slot.
